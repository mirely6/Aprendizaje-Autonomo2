Sistema de Compra de Productos de Electrovolt.
-Introducción-
Este proyecto fue desarrollado en Python con el objetivo de simular un proceso básico de compra dentro de la tienda Electrovolt.
El programa permite al usuario seleccionar un producto, indicar la cantidad que desea comprar, realizar el pago y
recibir una factura simple con el detalle de la compra y el cambio correspondiente. Durante el desarrollo se
aplicaron conceptos fundamentales de programamción como listas, tuplas, diccionarios, funciones, estructuras condicionales
y ciclos repetitivos.
-Objetivo del proyecto-
Crear un programa sencillo que permita gestionar una compra de productos, verificando la disponibilidad del artículo
seleccionado, calculando el valor total a pagar y vaalidando que el pago ingresado por el usuario sea suficiente sea 
suficiente para completar la transacción.
-Planificación del proyecto-
"Alcance del sistema"
El programa está diseñado para realizar las siguientes tareas:
* Mostrar los productos disponibles.
* Permitir al usuario seccionar un producto.
* Solicitar la cantidad deseada.
* Calcular automáticamente el total de la compra.
* Verificar que el pago sea suficiente.
* Calcular el cambio correspondiente.
* Mostrar una fractura básica al finalizar la compra.
-Problema a resolver-
En una venta es común tener que calcular el total de una compra, verificar el dinero recibido y determinar el cambio que debe entregarse al cliente.
Cuando estos cálculos se realizan manualmente pueden producirse errores. Este programa automatiza ese proceso de manera sencilla, permitiendo
realizar las operaciones de forma rápida y precisa.
-Funcionalidades Principales-
* Lista de productos.
Se utiliza una lista para almacenar y mostrar los productos disponibles para la venta.
* Tuplas.
Las tuplas permiten almacenar información que no necesita modificarse durante la ejecución del programa, como los precios base de los productos.
* Diccionarios.
Los diccionarios relacionan cada producto con su precio, facilitando la búsqueda y el cálculo del total de la compra.
* Funciones.
La lógica principal del programa se encuentra dentro de una función, lo que ayuda a mantener el código organizado y facilita su reutilización.
* Validación de productos.
El sistema verifica que el producto ingresado por el usuario exista dentro del catálogo antes de continuar con la compra.
* Cálculo del total.
Una vez seleccionada la cantidad, el programa calcula automáticamente el valor total a pagar.
* Validación del pago.
Si el monto ingresado es menor al total de la compra, el programa solicita nuevamente el pago hasta que sea suficiente.
* Cálculo del cambio.
Cuando el pago es válido, el sistema calcula y muestra el cambio que debe recibir el cliente.
* Generación de factura.
Al finalizar la compra se presenta un resumen con la información más importante de la transacción realizada.
